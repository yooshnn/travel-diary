# 유저 스토리

> 형식: 나는 [역할]로서, [기능]을 원한다.

---

## 공통 규칙

- 필수 입력값이 누락되거나 형식이 맞지 않으면 400을 반환한다
- 인증이 필요한 요청에서 토큰이 없거나 유효하지 않으면 401을 반환한다

---

## 인증 & 회원

### US-01 Google 로그인
**나는 비로그인 사용자로서, Google 계정으로 로그인하고 싶다.**

- [ ] Google OAuth 버튼 클릭 시 구글 로그인 페이지로 이동한다
- [ ] 로그인 성공 시 JWT를 발급하고 홈으로 리다이렉트한다
- [ ] 로그인 실패 시 로그인 페이지로 복귀한다

---

### US-02 로그아웃
**나는 로그인 사용자로서, 로그아웃하고 싶다.**

- [ ] 로그아웃 시 토큰이 삭제된다
- [ ] 홈으로 리다이렉트된다

---

### US-03 내 프로필 조회
**나는 로그인 사용자로서, 내 프로필을 조회하고 싶다.**

- [ ] 이름, 프로필 이미지 URL을 반환한다

---

### US-04 내 프로필 수정
**나는 로그인 사용자로서, 내 이름과 프로필 이미지를 수정하고 싶다.**

- [ ] 이름을 변경할 수 있다
- [ ] 이미지 파일(jpg, png 등)만 업로드 가능하다
- [ ] 수정 성공 시 변경된 정보를 반환한다

---

## 여행지

### US-05 여행지 검색 & 필터
**나는 사용자로서, 지도 범위 내 여행지를 검색·필터링하고 싶다.**

- [ ] 지도 범위(위경도 min/max)로 여행지를 조회한다
- [ ] 카테고리, 감성 태그, 키워드로 필터링할 수 있다
- [ ] 인기순(북마크 수), 이름순으로 정렬할 수 있다
- [ ] 커서 기반 페이징을 지원한다

---

### US-06 여행지 상세 조회
**나는 사용자로서, 여행지 상세 정보를 보고 싶다.**

- [ ] 이름, 사진, 설명, 주소, 좌표를 반환한다
- [ ] 등록자 정보, 북마크 수를 반환한다
- [ ] 카테고리, 대표 감정 태그를 반환한다
- [ ] 존재하지 않는 여행지 조회 시 404를 반환한다

---

### US-07 여행지 등록
**나는 사용자로서, 새 여행지를 등록하고 싶다.**

- [ ] 사진, 이름, 카테고리, 위치(위경도), 설명을 저장한다
- [ ] 설명은 10자 이상이어야 한다
- [ ] 등록 성공 시 생성된 ID를 반환한다

---

### US-08 홈 여행지 목록
**나는 사용자로서, 홈에서 인기·최신 여행지를 보고 싶다.**

- [ ] 인기순(북마크 수 기준) 목록을 반환한다
- [ ] 최신순(등록일 기준) 목록을 반환한다

---

## 일기 & 북마크

### US-09 일기 작성
**나는 사용자로서, 여행지에 일기를 작성하고 싶다.**

- [ ] 내용 10자 이상 필수다
- [ ] 사진(선택), 방문 날짜, 공개 여부를 저장한다
- [ ] 저장 시 `is_analyzed = false`로 초기화된다
- [ ] 방문 날짜는 오늘 혹은 그 이전이어야 한다

---

### US-10 내 일기 목록
**나는 사용자로서, 내 일기 목록을 날짜별로 보고 싶다.**

- [ ] 날짜 필터링을 지원한다
- [ ] 페이징을 지원한다
- [ ] 본인 일기만 조회된다

---

### US-11 일기 캘린더
**나는 사용자로서, 캘린더로 내 일기 활동 현황을 보고 싶다.**

- [ ] 연·월 기준으로 날짜별 일기 수를 반환한다

---

### US-12 여행지별 일기 목록
**나는 사용자로서, 여행지별 다른 사람들의 일기를 보고 싶다.**

- [ ] 공개 일기만 표시된다
- [ ] 본인의 비공개 일기는 본인에게만 표시된다
- [ ] 페이징을 지원한다

---

### US-13 홈 최신 일기
**나는 사용자로서, 최신 공개 일기를 홈에서 보고 싶다.**

- [ ] 최신 공개 일기 N개를 반환한다

---

### US-14 북마크 추가/삭제
**나는 사용자로서, 여행지를 북마크에 추가/삭제하고 싶다.**

- [ ] 북마크 추가·삭제를 토글할 수 있다

---

### US-15 내 북마크 목록
**나는 사용자로서, 내 북마크 목록을 보고 싶다.**

- [ ] 여행지 이름, 사진, 주소, 북마크 수를 반환한다
- [ ] 페이징을 지원한다

---

## AI 파이프라인

### US-16 일기 감정 분석 (시스템)
**나는 시스템으로서, 미분석 일기를 주기적으로 감정 분석하고 싶다.**

- [ ] `is_analyzed = false`인 일기만 처리한다
- [ ] LLM으로 감정을 분류하고 `diary_emotion_analysis`에 저장한다
- [ ] `attraction_emotion_stat`을 누적 갱신한다
- [ ] 완료 시 `is_analyzed = true`로 변경한다

---

### US-17 여행지 대표 감정 갱신 (시스템)
**나는 시스템으로서, 감정 통계를 바탕으로 여행지 대표 감정을 갱신하고 싶다.**

- [ ] `attraction_emotion_stat` 기반으로 베이지안 스코어를 계산한다
- [ ] 최고 스코어 감정으로 `attraction.emotion_id`를 갱신한다
- [ ] 갱신 여부와 무관하게 `ai_report`를 생성한다

---

### US-18 AI 분석 리포트 조회
**나는 사용자로서, 여행지의 AI 분석 리포트를 보고 싶다.**

- [ ] 해당 여행지의 최신 리포트를 반환한다
- [ ] 리포트가 없으면 404를 반환한다
- [ ] 분석 당시 대표 감정, 분석 내용, 생성 날짜를 포함한다
